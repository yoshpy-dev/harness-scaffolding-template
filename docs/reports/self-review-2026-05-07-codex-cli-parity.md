## Positive notes

- The `.claude/skills/` ↔ `.agents/skills/` mirror is genuinely byte-clean. `./scripts/check-skill-sync.sh` exits 0 with all 13 skills in lock-step on the current HEAD; the test harness in `tests/test-check-skill-sync.sh` exercises five drift modes (inventory, body, name, description, policy) plus a positive parity case. That is solid evidence the drift contract is enforceable, not just declarative.
- The Codex parity test (`internal/cli/cli_test.go:TestExecuteInit_RendersCodexSurfaces`) asserts on real init output rather than embed FS introspection, and it cross-checks the manifest tracks the new Codex-side paths (`.codex/config.toml`, `.agents/skills/spec/SKILL.md`) so a future `ralph upgrade` will see drift on those files. This is exactly the right shape — the test would catch a regression where `templates/base/.codex/` silently drops out of `go:embed` patterns.
- `internal/scaffold/embed_test.go:TestTemplateBaseCodexAssetsExist` guards the on-disk template even when `EmbeddedFS` is empty (which is the normal state in `go test ./...`). That sidesteps the "embed FS is zero-value in unit tests" trap that the existing `TestBaseFS_WithMockFS` had to skip around.
- `internal/cli/doctor.go:checkCodexEffectiveConfig` is appropriately conservative: it warns rather than fails on every soft condition (missing file, missing flag, no hooks) and only surfaces `fail` when the TOML itself is unparseable. That matches the plan's R-13 mitigation and keeps `ralph doctor` usable on bare-Claude projects.
- `internal/config/config_test.go:TestLoad_RequireCodexCLI` verifies the new TOML field round-trips both the explicit `true` and the explicit `false` direction. Round-trip tests on parsing knobs are easy to forget.
- `templates/base/.codex/AGENTS.override.md` and `templates/base/.codex/README.md` are short, opinionated, and explain *why* `/skill-name` syntax is forbidden (Codex built-in collision with `/plan`, `/review`, `/status`). That kind of "why" doc ages well.
- The rename from `codex-review` to `cross-review` was done as a single dedicated commit (`4d16323`) separate from the skill mirror commit (`1a1bc3d`). That is the right slicing — a future revert is feasible without churning the new `.agents/skills/` tree.
- `scripts/check-sync.sh` was updated with a comment block explaining why the old `docs/reports/codex-triage-` allowlist entry is kept ("Pre-rename historical triage reports kept for archival purposes only"). That comment will save someone an hour of "why is this line still here?" archaeology.
